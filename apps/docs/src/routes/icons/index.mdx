# Icons

Qwik Icons is a modern, build-time icon system that transforms JSX icon components into optimized SVG elements. The system uses a Vite plugin to convert `<Pack.Icon />` syntax into direct `jsx("svg", {...})` calls at build time.

## Key Features

- ✅ **Build-time transformation** - No runtime icon components
- ✅ **Type-safe** - Full TypeScript support with autocomplete
- ✅ **Optimized bundles** - Only used icons are included
- ✅ **A11y support** - Title props and custom children
- ✅ **Framework agnostic** - Works with any JSX runtime

## Supported Icon Packs

Our icon system includes icons from popular libraries:

- **Lucide** - Beautiful, consistent icon library
- **Heroicons** - Tailwind CSS icon library
- **Tabler** - Premium icon library
- **And more** - Any Iconify-supported pack

## Installation

Icons are included with the Qwik design system:

```bash
npm i @kunai-consulting/qwik
```

## Usage

Import and use icons directly in your components:

```tsx
import { component$ } from "@qwik.dev/core";
import { Lucide, Heroicons, Tabler } from "@kunai-consulting/qwik";

export const MyComponent = component$(() => {
  return (
    <div class="flex gap-4">
      <Lucide.Check width={24} class="text-green-500" />
      <Heroicons.Heart width={24} class="text-red-500" />
      <Tabler.Star width={24} class="text-yellow-500" />
    </div>
  );
});
```

## Icon Properties

Icons accept all standard SVG properties:

```tsx
<Lucide.Check
  width={24}
  height={24}
  class="text-blue-500"
  stroke-width={2}
  fill="currentColor"
/>
```

## Accessibility (A11y)

### Title Prop (Syntactic Sugar)

Use the `title` prop for simple accessibility:

```tsx
<Lucide.Info title="Information about this feature" width={24} />
```

This automatically generates:
```tsx
<svg ...>
  <title>Information about this feature</title>
  <!-- icon paths -->
</svg>
```

### Custom Children

For more complex accessibility or additional elements:

```tsx
<Lucide.User width={24}>
  <title>Profile</title>
  <desc>Click to view user profile</desc>
</Lucide.User>
```

## Styling

Icons inherit color and size from their parent:

```tsx
<div class="text-red-500 text-2xl">
  <Lucide.Heart /> {/* Will be red and 24px */}
</div>
```

You can also style individual icons:

```tsx
<Lucide.Check class="text-green-500 hover:text-green-700" width={32} />
```

## Build-time Transformation

The Vite plugin transforms icon JSX at build time:

**Input:**
```tsx
<Lucide.Check width={24} class="icon" />
```

**Output:**
```tsx
{jsx("svg", {
  width: 24,
  class: "icon",
  viewBox: "0 0 24 24",
  dangerouslySetInnerHTML: __qds_i_lucide_check
})}
```

## TypeScript Support

Full type safety with autocomplete:

```tsx
import { Lucide } from "@kunai-consulting/qwik";

// ✅ TypeScript knows this is Component<PropsOf<"svg">>
const checkIcon = <Lucide.Check />;

// ✅ All SVG props are available
<Lucide.Check
  width={24}
  height={24}
  class="custom-class"
  onClick$={handleClick}
  aria-label="Check mark"
/>
```

## Performance Benefits

- **Zero runtime** - Icons are transformed at build time
- **Tree-shaking** - Only used icons are included in bundles
- **Small bundles** - No icon component overhead
- **Fast rendering** - Direct SVG elements

## Migration from Old System

If you're migrating from the old `@kunai-consulting/qwik-icons`:

```tsx
// Old way
import { LuRocket } from "@kunai-consulting/qwik-icons";
<LuRocket />

// New way
import { Lucide } from "@kunai-consulting/qwik";
<Lucide.Rocket />
```

## Available Icon Packs

All Iconify-supported packs are available through type declarations. The most commonly used:

- `Lucide` - Modern, consistent icons
- `Heroicons` - Tailwind's icon library
- `Tabler` - Premium icon set
- `Material` - Google's Material Design icons
- And hundreds more...

## Custom Icon Packs

You can use any icon pack supported by Iconify. Just import the namespace and use the icons:

```tsx
import { CustomPack } from "@kunai-consulting/qwik";
<CustomPack.MyIcon />
```
