# Toast Research

## Overview
A Toast component provides brief, non-intrusive messages or notifications that appear temporarily on the screen.
The component follows WAI-ARIA alert pattern, ensuring accessibility while providing important feedback to users.

## Research Links:
- WAI-ARIA Alert Pattern (https://www.w3.org/WAI/ARIA/apg/patterns/alert/)
- Ark UI Toast (https://ark-ui.com/docs/components/toast)
- Kobalte Toast (https://kobalte.dev/docs/core/components/toast)
- Radix UI Toast (https://www.radix-ui.com/primitives/docs/components/toast)

## Component Structure

Pieces:
- Root (container and state management)
- Title (accessible heading)
- Description (message content)
- Close (dismissal button)

---

## Keyboard Interactions
- Escape: Dismiss active toast
- Tab: Navigate through interactive elements
- Enter/Space: Activate focused interactive element

---

## Attributes
- role="alert" (root)
- role="status" (for non-critical messages)
- aria-live="polite" (default) or "assertive" (for important messages)
- aria-labelledby (references title)
- aria-describedby (references description)
- data-qds-toast-root
- data-qds-toast-title
- data-qds-toast-description
- data-qds-toast-close
- data-open
- data-closed
- data-swipe-direction
- data-position
